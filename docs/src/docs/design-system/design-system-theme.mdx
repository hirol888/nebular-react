# Eva Design System Theme

In Eva Design System a **theme** is a set of semantic variables and connections between them, that represents the application's look & feel to achieves the following goals:

- create new visual themes easily;
- flexibly change look & feel of the application by managing variables, without changing components' styles;
- switch between visual themes in app runtime without page reload;
- support of CSS-IN-JS.

<hr />

## A Theme

In Nebular terms - **theme** is a javascript object, structured in a particular way. Here's a gist of how it looks like:

```ts
export const DefaultThemeCore: NebularThemeCore = {
  theme_name: 'default',
  isDarkTheme: false,
  color_primary_100: constants.color_primary_100,
  color_primary_200: constants.color_primary_200,
  color_primary_300: constants.color_primary_300,
  color_primary_400: constants.color_primary_400,
  color_primary_500: constants.color_primary_500,
  color_primary_600: constants.color_primary_600,
  color_primary_700: constants.color_primary_700,
  color_primary_800: constants.color_primary_800,
  color_primary_900: constants.color_primary_900
  // ...
};
```

Each theme is divided into the following semantic groups:

- Colors
- Backgrounds & Borders
- Text Colors
- Fonts & Text Styles
- General Theme Variables

<hr />

## Colors

All available color within the theme. 5 semantic colors (`primary`, `success`, `info`, `warning`, `danger`), 6 transparency levels for every default semantic color (8%, 16%, 24%, 32%, 40%, 48%) and `basic` color (backgrounds and texts).
Each color has a pallet of 9 shades, except for `basic`, which has 11 shades. These colors mostly used by `status` variants of the components.

Primary color shades:

```ts
export const color_primary_100 = '#f2f6ff';
export const color_primary_200 = '#d9e4ff';
export const color_primary_300 = '#a6c1ff';
export const color_primary_400 = '#598bff';
export const color_primary_500 = '#3366ff';
export const color_primary_600 = '#274bdb';
export const color_primary_700 = '#1a34b8';
export const color_primary_800 = '#102694';
export const color_primary_900 = '#091c7a';

export const color_primary_transparent_100 = 'rgba(51, 102, 255, 0.08)';
export const color_primary_transparent_200 = 'rgba(51, 102, 255, 0.16)';
export const color_primary_transparent_300 = 'rgba(51, 102, 255, 0.24)';
export const color_primary_transparent_400 = 'rgba(51, 102, 255, 0.32)';
export const color_primary_transparent_500 = 'rgba(51, 102, 255, 0.4)';
export const color_primary_transparent_600 = 'rgba(51, 102, 255, 0.48)';
```

You can also tune colors used for element states:

```ts
export const color_primary_focus = 'var(--color-primary-600)';
export const color_primary_hover = 'var(--color-primary-400)';
export const color_primary_default = 'var(--color-primary-500)';
export const color_primary_active = 'var(--color-primary-600)';
export const color_primary_disabled = 'var(--color-basic-transparent-300)';

//...

  color_primary_focus: constants.color_primary_focus,
  color_primary_hover: constants.color_primary_hover,
  color_primary_default: constants.color_primary_default,
  color_primary_active: constants.color_primary_active,
  color_primary_disabled: constants.color_primary_disabled,
```

You can adjust these settings for each of the colors, to make states use lighter or darker colors.

## Backgrounds & Borders

A theme has 3 backgrounds (`basic`, `alternative`, `primary`) each of 4 shades and also 3 borders, each of 5 shades (background shades count + 1).
`basic` and `alternative` backgrounds and borders utilize `basic` color shades as a source. `primary` backgrounds and borders use `primary` color.
Basic backgrounds and borders heavily used by components (cards, accordions, menu, etc), when alternative (tooltips) and primary only for particular use cases,
to distinguish some of the components.

Basic backgrounds and borders:

```ts
export const background_basic_color_1 = 'var(--color-basic-100)';
export const background_basic_color_2 = 'var(--color-basic-200)';
export const background_basic_color_3 = 'var(--color-basic-300)';
export const background_basic_color_4 = 'var(--color-basic-400)';

export const border_basic_color_1 = 'var(--color-basic-100)';
export const border_basic_color_2 = 'var(--color-basic-200)';
export const border_basic_color_3 = 'var(--color-basic-300)';
export const border_basic_color_4 = 'var(--color-basic-400)';
export const border_basic_color_5 = 'var(--color-basic-500)';

// ...

  background_basic_color_1: constants.background_basic_color_1,
  background_basic_color_2: constants.background_basic_color_2,
  background_basic_color_3: constants.background_basic_color_3,
  background_basic_color_4: constants.background_basic_color_4,

  border_basic_color_1: constants.border_basic_color_1,
  border_basic_color_2: constants.border_basic_color_2,
  border_basic_color_3: constants.border_basic_color_3,
  border_basic_color_4: constants.border_basic_color_4,
  border_basic_color_5: constants.border_basic_color_5,
```

the most used of these are:

- `background_basic_color_1` - the lightest one, usually used for top sitting elements - cards, headers, etc.
- `background_basic_color_2` - for background of the layout and input controls (inputs, checkboxes, etc)

And vice-versa for dark themes:

```ts
  background_basic_color_1: 'var(--color-basic-800)',
  background_basic_color_2: 'var(--color-basic-900)',
  background_basic_color_3: 'var(--color-basic-1000)',
  background_basic_color_4: 'var(--color-basic-1100)',

  border_basic_color_1: 'var(--color-basic-800)',
  border_basic_color_2: 'var(--color-basic-900)',
  border_basic_color_3: 'var(--color-basic-1000)',
  border_basic_color_4: 'var(--color-basic-1100)',
  border_basic_color_5: 'var(--color-basic-1100)',
```

## Text Colors

There are 5 colors within the theme: `basic` - main text color, used on top of `basic` backgrounds `alternate` - alternative color used on top of `alternate` backgrounds,
`control` - used on top of `status` colors (`primary`, `success`, etc), `disabled` color - to indicate text/component disabled state
and `hint` - for secondary texts (for example placeholders and captions).

Text colors use `basic` shades as a source:

```ts
export const text_basic_color = 'var(--color-basic-800)';
export const text_alternate_color = 'var(--color-basic-100)';
export const text_control_color = 'var(--color-basic-100)';
export const text_disabled_color = 'var(--color-basic-transparent-600)';
export const text_hint_color = 'var(--color-basic-600)';

// ...

  text_basic_color: constants.text_basic_color,
  text_alternate_color: constants.text_alternate_color,
  text_control_color: constants.text_control_color,
  text_disabled_color: constants.text_disabled_color,
  text_hint_color: constants.text_hint_color,
```

<hr />

## Fonts & Text Styles

Each theme has two available fonts: `default` and `secondary`. `secondary` font used for headers, while the default for the rest of the elements.

There are 14 text styles:

- 6 `heading` styles, used by h1-h6 elements
- 2 `subtitle` styles, used as a text of most of controls (inputs, menus, etc)
- 2 `paragraph` styles, regular text
- 2 `caption` styles, used by smaller texts, like tooltip or input caption
- `label` style, used by label element
- `button` text style, used by button element

Each styles describes text `font-family`, `font-size`, `font-width` and `line-height`, for instance, caption text style:

```ts
export const text_caption_font_family = 'var(--font-family-primary)';
export const text_caption_font_size = '0.75rem';
export const text_caption_font_weight = 400;
export const text_caption_line_height = '1rem';

// ...

  text_caption_font_family: constants.text_caption_font_family,
  text_caption_font_size: constants.text_caption_font_size,
  text_caption_font_weight: constants.text_caption_font_weight,
  text_caption_line_height: constants.text_caption_line_height,
```

<hr />

Adjust these styles to change text style of specific groups of elements.
